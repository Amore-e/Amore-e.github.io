# 链表

链表通常是由一连串的节点组成的，每个节点包含任意类型的实例数据和一个或者两个用来指向下一个或上一个节点位置的链接（指针）。
链表是一种线性表，但是并不会按照线性的顺序存储数据，而是在每一个节点存放下一个节点的指针。

# 单链表

  • 一个单链表的节点组成分为两个部分，第一部分保存节点的数据，第二个部分保存下一个节点的地址。最后一个节点存储地址的部分指向空指针。
  • 单链表只可以向一个方向遍历。查找一个节点的时候，都需要从第一个节点开始，每次访问下一个节点，一直到目标位置。

单链表和数组

  • 存储空间：数组的存储空间是连续的，链表的存储空间是不连续的。
  • 长度可变性：链表的长度可以根据实际需要，自定伸缩。数组长度的长度一旦定义就不能改变。
  • 数据访问：链表的访问，需要一定访问，数组访问更加便捷。
  • 使用场景：数据量大，而且需要频繁的访问数据时，数组更适合。频繁需要插入和删除操作的链表更适合。

  • 运算规则：
  • 遍历：数组是连续的，所以遍历效率高；链表必须从头开始，遍历效率低。
  • 插入和删除：数组的操作，需要移位（效率高），链表只需要改变指针的指向。

栈、队列、链表的区别

  • 栈、队列是线性表，操作受限。区别在于运算规则不同。
