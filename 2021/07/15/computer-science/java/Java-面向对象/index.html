# 面向对象

# 面向对象

表格思维就是一种典型的面向对象的思维。

面向对象和面向过程的区别 ：

  • 面向对象具有三大特征：封装性、继承性和多态性，而面向过程没有继承性和多态性，并且面向过程的封装只是封装功能，而面向对象可以封装数据和功能。所以面向对象优势更明显。

  • 一个经典的比喻：面向对象是盖浇饭、面向过程是蛋炒饭。盖浇饭的好处就是 “菜”“饭” 分离，从而提高了制作盖浇饭的灵活性。饭不满意就换饭，菜不满意换菜。用软件工程的专业术语就是“可维护性”比较好，“饭” 和 “菜” 的耦合度比较低。

# 类和对象

面向对象的两大核心：

  • 类 ：是对现实生活中一类具有共同属性的行为的事物的抽象。

    • 类的特点：
      • 类是对象的数据类型。
      • 类是具有相同属性和行为的一组对象的集合。

  • 对象 ：类的具体化。

    • 属性：对象具有的各种特种，每个对象的每个属性拥有特定的值。
    • 行为：对象能够执行的操作。

  • 类和对象的关系 ：

    ​ 类 ，是对现实生活中一类具有共同属性的行为的事物的抽象。
    ​ 对象，就是该类下的一个真实存在的实体。

类

  • 类的重要性：是 java 程序的基本组成单元。类可以看作是一类对象的模板，对象可以看作是该类的一个具体实例。类是用于描述同一类型的对象的抽象概念，类中定义了这一类对象所具有的共同属性和方法。

  • 类的组成：

    • 成员变量： 定义在类的内部，方法的外部的变量。体现类的属性。
    • 成员方法：体现类的行为。

  • 类的定义步骤：

    1. 定义一个类。
    2. 编写类的成员变量。
    3. 编写类的成员方法。

对象

  • 创建对象：类名（类型） 对象名（变量）= new 类名 ();
  • 使用对象：
    • 使用成员变量：
      • 对象名。属性名；// 获取属性
      • 对象名。属性名 = 对应类型的值；// 给属性赋值
    • 使用成员方法：
      • 格式 对象名。方法名（实参列表）

对象的内存分析

Java 中的类，属于引用数据类型。引用数据类型的内存分析，涉及两块常见的内存空间：

  • 堆内存：保存对象的具体信息。在程序中通过 new 来开辟堆内存。
  • 栈内存：保存一块堆内存的地址。(通过地址找到堆内存，然后找到对象内容)。

对象的实例化有两种方式：

  • 声明并完成实例化对象。（一步到位）
  • 先声明，再实例化。（分步进行）
  1. new 类() 的作用是派皮堆内存，而将他赋值给一个变量，本质上是将堆内存的地址存在了栈内存中。
  2. 将对象进行信息修改时，先通过栈内存中存放的地址找到堆内存中的具体信息，然后对对象信息进行修改。
  3. 如果程序只声明了对象，但未进行实例化，此时无法调用对象信息。常常会出现 空指针异常 。
  4. 多个对象 （同类）在内存中，都有不同的内存划分。成员变量存储在各自的内存区域中，成员方法多个对象共有的。
  5. 当多个对象的引用指向相同的内存空间时（即，多个变量记录的对内存地址相同）：只要任何一个对象修改了内存中的数据，之后任何一个对象去访问数据，都是修改后的数据。

 

# 成员变量和局部变量

  • 存在位置：
    • 成员变量位于类的内部，方法的外部；
    • 局部变量位于方法的内部 / 形参。
  • 内存中的位置：
    • 成员变量位于堆内存；
    • 局部变量位于栈内存；
  • 生命周期：
    • 成员变量随着对象的产生而产生，随着对象的消亡而消亡。
    • 局部变量随着方法的调用而产生，随着方法的执行结束而消失。
  • 初始值：
    • 成员变量有默认的初始值。
    • 局部变量没有默认值。必须先定义再使用，并且在使用之前必须进行显示的初始化操作。

# 状态修饰符

final

final 是一个关键字，他可以修饰成员变量、成员方法、类。
final 修饰的特点：

  • 修饰方法：修饰的方法，不能被重写；
  • 修饰成员变量：修饰的变量是常量。不能被二次赋值，只能赋值一次。
  • 修饰类：修饰的类不能被继承。
  • 修饰局部变量：被修饰的变量是基本类型时，变量的值不能被改变。被修饰的是引用类型时，变量的地址值无法改变，但是引用地址指向的对象的属性是可以改变的。

static

static 可以修饰成员方法、成员变量。在 java 类中，可以使用 static 修饰属性、方法、代码块、内部类。

被 static 修饰的成员的特点：

  • 随着类得加载而加载，优先于对象存在；
  • 修饰的成员，被所有对象所共享。
  • 访问权限允许时，可以不创建对象，直接被类调用。

static 的访问特点：

  • 非静态成员方法：能访问静态的成员变量、成员方法，也能访问非静态的成员变量和方法。
  • 静态成员方法：只能访问静态的成员变量和成员方法。

 

# 方法

方法的定义和调用

  • 无参方法的定义和调用

每个方法在被调用执行的时候，都会被加载到栈内存，并且拥有自己独立的内存空间，方法内部的代码执行完毕之后，会从栈内存中出栈 ，释放空间。

  • 带参方法的定义和调用

调用：

方法调用时，传入的参数的数量和类型必须和方法定义中的类型和数量相匹配。否则 程序就会报错。

形参：方法定义是的参数。
实参：方法调用时传递进来的参数。

形参和实参的功能就是传递数据，发生方法调用时， 实参的值会传递给形参。

形参和实参的区别和联系:

  1. 形参变量只有在方法被调用时才会分配内存，调用结束后 ，就会被立即释放。所以形参变量只在方法内部有效，不能再方法的外部使用。
  2. 实参可以是常量 、变量、表达式、方法等。无论实参是何种类型的数据，在进行方法的调用时，都必须有确定的值。
  3. 实参和形参在数量上、类型上、顺序上都必须严格一致。当然在其中，能够自动类型转化的除外。
  4. 方法调用中，发生的数据的传递是单向。只能有实参传递给形参，而不能把形参反向传递给实参。一旦参数的传递完成，则实参和形参将不会有任何的关系。在方法的调用过程中，形参的值的改变不会影响实参。
  • 带返回值的方法的定义和调用

注意事项：

  • 方法不能嵌套定义。方法和方法之间是平行关系，方法和方法之间只能存在调用关系。
  • void 表示无返回值，可以省略 return。或者 return ; 。
  • 方法必须被调用才能执行，不调用是不会执行的。

方法的参数传递

  • 值传递：传递的数据本身 。
  • 引用传递：传递的地址 传递的是一个引用 所以引用的只要是同一个地址空间，那么任何地方对空间内容的修改，都将引起整个的变化。

方法的重载

构成方法重载的条件：

  • 形参的个数不同
  • 形参类型不同
  • 不同类型的参数，顺序不同也可以构成重载

特别注意： 方法的返回值类型，不能作为方法是否重载的判断依据。

 

# 构造方法

构造方法也称为构造器（constructor），用于对象的初始化。构造方法是一个创建对象时被自动调用的特殊方法。

构造方法的名称要与类名完全一致。构造方法没有返回值（void 也不写）。Java 通过 new 关键字来调用构造方法，从而返回该类的一个实例对象，是一种特殊的方法。

构造方法的注意事项

  • 如果没有自定义构造方法，系统将会给出一个默认的无参构造方法。如果自定义了构造方法，系统将不再提供默认的无参构造器，若还需要使用无参构造器，就必须手动添加。
  • 无论是否使用无参构造，在开发中，都推荐显式定义无参构造。
  • 构造方法的一个重要作用是使用带参构造器为成员变量进行初始化。
  • 构造方法的调用：在构造方法中可以调用其他的构造方法。可以使用 this参数列表） 调用本类的其它构造方法。可以使用 super(参数列表) 调用父类的构造方法。需要注意的一点是，使用 this 和 super 调用构造方法。必须位于构造方法的第一条语句。因此对于构造方法的调用，只能调用一次。

 

# Java Bean

标准Java Bean的写法

  1. 成员变量：使用 private 修饰。
  2. 提供 getter 和 seter 方法，来访问成员变量。
  3. 提供构造方法： 无参的和带有多个参数的。
